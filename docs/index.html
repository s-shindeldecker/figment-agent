Figment
E100 Experimentation Targeting — How It Works
Internal
April 2026

Figment is the automated agent that rebuilds and maintains the E100 experimentation target list — the 100 highest-priority accounts for the Experimentation Solutions team. It replaces a manual, stale spreadsheet with a live, multi-source scoring system that refreshes weekly without a human owner.

What changed
Before — manual E100
Single Snowflake export, refreshed manually by one owner
Last data update: 105+ weeks ago when maintained
Prospects added by gut feel and rep memory
No competitive intelligence signals
Lost momentum entirely when the owner left
Single undifferentiated list, no tier logic
Now — Figment
Three data sources, automated weekly refresh
Looker live query — always current month's usage data
Enterpret Cypher queries surface competitor signals from 12K+ Gong calls
ZoomInfo technographic + scoop intent for cold prospects
Survives team changes — no single human owner required
Three scored tiers with distinct motions and criteria

Architecture
Pipeline
Deterministic Python batch (asyncio). Same inputs + same config = same ranked output. No LLM in the scorer.
Configuration
Prompts and Cypher queries live in YAML — wisdom_cypher.yaml, settings.yaml, tier3_sources.yaml. Update without touching code.
Data model
Single AccountRecord type (core/schema.py). Two parallel views: per-tier exports for Sheets, merged list for console and Slack.
Output
Google Sheets (3 tier tabs + optional merged master + changelog), Slack top-N weekly digest, run-to-run diff via local snapshot.
Human override layer preserved — AEs, CSMs, and Exp Solutions can flag any account for inclusion, exclusion, or escalation.
